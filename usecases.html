Cross-Industry Patterns

Before diving into specific industries, here are the five agent archetypes you'll see across every vertical. Each maps to a specific set of Okta capabilities.

Archetype 1

Customer-Facing Agents

Agents that interact with end customers.

Primary: Token Vault + CIBA

Industries: FinServ, Retail, Insurance, Healthcare

Risk: Agent accesses all customer data regardless of authorization.

Archetype 2

Internal Knowledge Agents

Search and synthesize internal docs.

Primary: FGA + RAG Authorization

Industries: Technology, Government, Manufacturing

Risk: Confidential docs leak into any employee's LLM context.

Archetype 3

Compliance & Audit Agents

Monitor regulations, generate reports.

Primary: OIG + System Logs + ISPM

Industries: FinServ, Healthcare, Insurance, Gov

Risk: Compliance agent becomes ungoverned access point.

Archetype 4

Multi-System Orchestration

Coordinate across SaaS + internal APIs.

Primary: XAA + MCP + Token Vault

Industries: Technology, Manufacturing, Retail

Risk: Single compromise cascades across all systems.

Archetype 5

Human-Approved Workflows

High-value actions requiring sign-off.

Primary: CIBA + Delegation Chain

Industries: FinServ, Healthcare, Insurance, Gov

Risk: Agent auto-approves wire transfers, medication changes, etc.

Fraud Detection Agent

Persona: Maria

Fraud analyst at mid-size bank. Monitors 50,000+ daily transactions. Currently experiences 60% false positive rate.

The Problem

Bank deploys AI fraud agent. Without identity, agent has service account with read access to ALL transactions. Compromised via supply chain attack → attacker accesses every transaction. Audit log shows "svc-fraud."

Secure Workflow (7 Steps)

1

Maria logs in via Okta SSO with MFA.

2

Asks agent: "Analyze unusual patterns in high-value wire transfers this week."

3

Agent authenticates with certificate (Agent Identity).

4

Token exchange: credentials + Maria's ID token → scoped ID-JAG.

5

Queries transaction DB — only wire transfers, only Maria's accounts, only this week.

6

Returns analysis. Maria flags 3 transactions.

7

CIBA request: "Freeze account #4521 pending investigation" → Maria approves on phone.

Required Capabilities

Agent Identity XAA FGA CIBA System Logs

Before & After

Without Okta

Service account → full DB → compromise = all transactions → "svc-fraud" in logs

With Okta

Delegation → scoped to analyst's accounts → compromise = 5-min window → "agent:fraud-detector acting as maria@bank.com"

Impact

  • 50% reduction in false positives
  • Blast radius reduced from "all transactions" to "5-minute window"
SE ONLY

Seller Mode: Blast-Radius Advantage

Lead with blast-radius comparison. CISOs understand risk reduction. If "we already have fraud AI" → "Who governs its access? Can you prove unauthorized access was blocked?"

Wealth Advisory Agent

Persona: James

Wealth advisor managing 200 high-net-worth portfolios. Spends 40% of day pulling data from 6 different systems.

The Problem

AI assistant aggregates 6 systems. Agent stores James's credentials in config. Vacation → another advisor queries client estate plan → audit shows James accessed it.

Secure Workflow (7 Steps)

1

James asks: "Prepare quarterly review for client Chen Wei."

2

Agent authenticates with certificate.

3

Token exchange → scoped ID-JAG.

4

Token Vault: Retrieve tokens for Salesforce (contacts), Bloomberg (market), internal API (holdings).

5

FGA check: Can James view Chen Wei's estate planning? → Yes (assigned advisor).

6

Compiles package. James reviews and sends.

7

Vacation → delegation shifts to covering advisor, scoped to their clients.

Required Capabilities

Token Vault XAA FGA Agent Identity

Before & After

Without Okta

Stored credentials → all clients → wrong audit trail → leak = 200 portfolios

With Okta

Delegation → scoped to advisor's clients → proper handoff → Token Vault manages tokens

Impact

  • 40% less time gathering data across 6 systems
  • Zero cross-client exposure

Regulatory Compliance Agent

Persona: Priya

Head of Compliance managing team of 5. 200+ regulatory updates per month. Currently 3 months behind on processing.

The Problem

AI scans regulatory feeds. Without governance, stores analysis in shared drive. Junior analyst finds draft enforcement assessment — restricted to compliance committee.

Secure Workflow (7 Steps)

1

Priya asks: "OCC guidance changes affecting commercial lending, past 30 days."

2

Agent authenticates with certificate.

3

Accesses regulatory APIs via XAA (scoped to commercial lending).

4

RAG search of internal policies — FGA scopes to Priya's authorized policies.

5

Generates draft impact assessment.

6

Stores in compliance-restricted folder — FGA enforces committee access only.

7

OIG quarterly: "Should agent still access enforcement-sensitive docs?" → Priya confirms.

Required Capabilities

XAA FGA OIG System Logs

Before & After

Without Okta

Shared drive → anyone sees enforcement drafts → no review → years of stale access

With Okta

FGA-protected storage → committee only → quarterly OIG → access audited

Impact

  • 80% faster regulatory processing
  • Zero unauthorized access
SE ONLY

Seller Mode: Compliance Traceability

Compliance Officer: "EU AI Act Article 12 requires traceability." OIG certification is the differentiator. Position as regulatory enablement, not overhead.
